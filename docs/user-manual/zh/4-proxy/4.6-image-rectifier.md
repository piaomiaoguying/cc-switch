# 4.6 图片整流器

## 概述

图片整流器是代理层的功能，在请求转发前自动检测并替换 API 请求中的 base64 图片块。

**解决的问题：** 大量第三方 API 供应商不支持多模态（视觉理解）能力。当 messages 数组中出现包含 base64 数据的 `type: "image"` 块时，这些供应商会报错或行为异常。

**解决方案：** 整流器在请求转发之前拦截，扫描所有 `messages[*].content` 数组，找到 `type: "image"` 块，移除 base64 数据，替换为文本提示，引导模型调用指定 skill 来处理图片。

## 启用方式

1. 打开 CC Switch，进入**设置**页面。
2. 切换到**代理**选项卡。
3. 滚动到**高级整流器**区域。
4. 找到**图片整流器**，打开开关。
5. 可选：修改 **Skill 名称**，指定模型应调用的 skill（默认：`image-analysis`）。

## 配置项

| 设置 | 说明 | 默认值 |
|------|------|--------|
| 启用图片整流 | 功能总开关 | 关闭 |
| Skill 名称 | 提示模型调用的 skill 标识名 | `image-analysis` |

## 工作原理

1. 代理在转发请求前检查图片整流是否已启用。
2. 遍历请求体中每条消息的 `content` 数组。
3. 对每个 `type` 为 `"image"` 的块：
   - 统计移除的 base64 数据字符数
   - 检查后一个块是否为图片缓存引用（`[Image: source: /path/to/file]`）
   - 将 image 块替换为文本块：`"[Image: 此处有一张图片，请使用 '{skill_name}' skill 读取并分析这张图片。图片路径: /path/to/file.png]"`
4. 修改后的请求随即转发给上游供应商。

如果没有找到 `type: "image"` 块，请求原样通过。

## 与图片分析技能的配合

整流器设计为与[图片分析技能](./4.7-image-analysis-skill.md)配合使用。同时启用时：

1. 整流器移除图片数据，插入提示让模型调用 `image-analysis`
2. 模型收到文本提示，执行 `image-analysis` CLI 命令
3. 技能使用预配置的视觉 AI provider 处理实际图片文件（或剪贴板数据）
4. 模型收到视觉分析结果，继续响应

## 注意事项

- 图片整流仅影响通过代理转发的请求，直连 API 调用不受影响。
- 默认关闭以避免干扰支持多模态的供应商。
- base64 图片数据可达数 MB，移除后显著减小请求体体积。
