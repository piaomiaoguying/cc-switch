# 4.7 图片分析技能

## 概述

图片分析技能（`image-analysis`）是一个独立的 Python 命令行图片视觉识别工具，支持 30+ AI provider，兼容任何能执行 Shell 命令的 AI CLI。

## 安装与配置

技能位于项目根目录的 `skills/image-analysis/` 下。

```bash
# 复制配置模板
cp skills/image-analysis/scripts/config.example.json skills/image-analysis/scripts/config.json

# 编辑 config.json，填入你的 API key
```

`config.json` 文件结构：

```json
{
  "default_model": "zhipu",
  "providers": {
    "provider-name": {
      "api_key": "你的API密钥",
      "model": "模型ID",
      "base_url": "https://api.example.com/v1/chat/completions"
    }
  }
}
```

`config.json` 文件已被 gitignore，不会被提交。

## 使用方式

```bash
# 命令格式
python skills/image-analysis/scripts/vision.py analyze [选项]
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `--image 路径/URL` | 图片文件路径或 URL。可多次指定用于对比。不传时自动读取 macOS 剪贴板。 |
| `--prompt 文本` | 分析提示词（必填） |
| `--model 名称` | 指定使用某个 provider |
| `--fallback` | 按配置顺序依次尝试所有 provider，失败自动切换 |
| `--thinking` | 开启思考模式（需 provider 支持） |
| `--json` | JSON 格式输出，含 provider 名称 |
| `--show-usage` | 显示 token 用量（输入/输出/总计） |
| `--config 路径` | 指定自定义 config.json 路径 |

### 使用示例

```bash
# 本地图片 fallback 分析
python skills/image-analysis/scripts/vision.py analyze \
  --image screenshot.png \
  --prompt "这张截图中显示了什么错误信息？" \
  --fallback

# 网络图片分析
python skills/image-analysis/scripts/vision.py analyze \
  --image https://example.com/photo.jpg \
  --prompt "描述这张照片" \
  --model zhipu

# 两张图片对比
python skills/image-analysis/scripts/vision.py analyze \
  --image before.png \
  --image after.png \
  --prompt "这两张截图有什么变化？" \
  --fallback --json

# 剪贴板图片 + 思考模式
python skills/image-analysis/scripts/vision.py analyze \
  --prompt "读取这张图片中所有文字" \
  --thinking --fallback --show-usage
```

## 支持格式

- **本地文件：** jpg、jpeg、png、gif、webp、bmp（根据扩展名自动识别）
- **网络 URL：** 任何可公开访问的图片 URL
- **剪贴板（macOS）：** 通过 AppleScript 读取系统剪贴板中的 PNG/TIFF/GIF

## 与图片整流器的配合

[图片整流器](./4.6-image-rectifier.md)与此技能协同工作：

1. 非视觉模型收到含图片块的请求
2. 整流器移除图片，插入提示：`"[Image: 此处有一张图片，请使用 'image-analysis' skill 读取并分析这张图片。图片路径: /tmp/photo.png]"`
3. 模型读取提示，生成 Shell 命令调用 `vision.py analyze`
4. 图片分析技能调用配置的视觉 AI provider 完成分析
5. 视觉分析结果返回模型，模型继续处理

这条链路让非视觉模型可以有效处理基于图片的查询，无需修改上游供应商。

## 注意事项

- 推荐使用 `--fallback` 提高可靠性：某个 provider 不可用时自动切换下一个。
- 剪贴板功能仅在 macOS 上可用，且无需传 `--image` 参数。
- `config.json` 包含 API key，已被 gitignore，切勿提交。
- 所有 provider 均使用 OpenAI 兼容的 `/v1/chat/completions` 端点。
