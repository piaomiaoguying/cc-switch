# 4.7 画像分析スキル

## 概要

画像分析スキル（`image-analysis`）は、30 以上の AI プロバイダーに対応するスタンドアロンの Python CLI 画像視覚認識ツールです。シェルコマンドを実行できる任意の AI CLI と互換性があります。

## インストールと設定

スキルはプロジェクトルートの `skills/image-analysis/` にあります。

```bash
# 設定テンプレートをコピー
cp skills/image-analysis/scripts/config.example.json skills/image-analysis/scripts/config.json

# config.json を編集して API キーを追加
```

`config.json` の構造：

```json
{
  "default_model": "zhipu",
  "providers": {
    "provider-name": {
      "api_key": "あなたのAPIキー",
      "model": "モデルID",
      "base_url": "https://api.example.com/v1/chat/completions"
    }
  }
}
```

`config.json` は gitignore されており、コミットされません。

## 使い方

```bash
# コマンド形式
python skills/image-analysis/scripts/vision.py analyze [オプション]
```

### オプション

| オプション | 説明 |
|--------|-------------|
| `--image パス/URL` | 画像ファイルパスまたは URL。比較のために複数回指定可能。省略時は macOS クリップボードから読み取り。 |
| `--prompt テキスト` | 分析プロンプト（必須） |
| `--model 名前` | 特定のプロバイダーを使用 |
| `--fallback` | 設定された順序ですべてのプロバイダーを試行し、失敗時に自動切替 |
| `--thinking` | 思考モードを有効化（プロバイダー対応時） |
| `--json` | プロバイダー名付き JSON 形式で出力 |
| `--show-usage` | トークン使用量（入力/出力/合計）を表示 |
| `--config パス` | カスタム config.json へのパス |

### 使用例

```bash
# フォールバックモードでローカル画像を分析
python skills/image-analysis/scripts/vision.py analyze \
  --image screenshot.png \
  --prompt "このスクリーンショットにどんなエラーメッセージが表示されていますか？" \
  --fallback

# ネットワーク画像を分析
python skills/image-analysis/scripts/vision.py analyze \
  --image https://example.com/photo.jpg \
  --prompt "この写真を説明してください" \
  --model zhipu

# 2 つの画像を比較
python skills/image-analysis/scripts/vision.py analyze \
  --image before.png \
  --image after.png \
  --prompt "これら 2 つのスクリーンショットの違いは？" \
  --fallback --json

# クリップボード画像 + 思考モード
python skills/image-analysis/scripts/vision.py analyze \
  --prompt "この画像のすべての文字を読み取ってください" \
  --thinking --fallback --show-usage
```

## 対応フォーマット

- **ローカルファイル：** jpg、jpeg、png、gif、webp、bmp（拡張子で自動判別）
- **ネットワーク URL：** 公開アクセス可能な任意の画像 URL
- **クリップボード（macOS）：** AppleScript 経由でシステムクリップボードの PNG/TIFF/GIF を読み取り

## 画像整流器との連携

[画像整流器](./4.6-image-rectifier.md)とこのスキルは連携して動作します：

1. 非ビジョンモデルが画像ブロックを含むリクエストを受信
2. 整流器が画像を削除し、`"[Image: この画像を読み取って分析するには 'image-analysis' スキルを使用してください。画像パス: /tmp/photo.png]"` を挿入
3. モデルがヒントを読み取り、`vision.py analyze` を呼び出すシェルコマンドを生成
4. 画像分析スキルが設定された視覚 AI プロバイダーで分析を実行
5. 視覚分析結果がモデルに返され、処理を継続

このチェーンにより、上流プロバイダーを変更することなく、非ビジョンモデルが画像ベースのクエリを効果的に処理できます。

## 注意事項

- 信頼性のために `--fallback` フラグの使用を推奨：一つのプロバイダーがダウンしても自動的に次を試行。
- クリップボード機能は macOS のみ対応で、`--image` 引数は不要。
- `config.json` には API キーが含まれ、gitignore されています。絶対にコミットしないでください。
- すべてのプロバイダーは OpenAI 互換の `/v1/chat/completions` エンドポイントを使用。
