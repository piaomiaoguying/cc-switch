# 4.6 Image Rectifier

## Overview

The Image Rectifier is a proxy-layer feature that automatically detects and replaces base64 image blocks in API requests before they are forwarded to the upstream provider.

**Problem:** Many third-party API providers do not support multimodal (vision) capabilities. When a `type: "image"` block with base64 data appears in the messages array, these providers return errors or behave unexpectedly.

**Solution:** The rectifier intercepts the request before forwarding, scans all `messages[*].content` arrays, finds `type: "image"` blocks, removes the base64 data, and replaces them with a text prompt instructing the model to call a designated skill to handle the image.

## Enabling the Image Rectifier

1. Open CC Switch and go to **Settings**.
2. Navigate to the **Proxy** tab.
3. Scroll down to **Advanced Rectifier**.
4. Find **Image Rectifier** and toggle the switch to enable it.
5. Optionally, change the **Skill Name** field to specify which skill the model should call (default: `image-analysis`).

## Configuration

| Setting | Description | Default |
|---------|-------------|---------|
| Enable Image Rectification | Master switch for the feature | Off |
| Skill Name | The skill identifier the model will be prompted to call | `image-analysis` |

## How It Works

1. Before forwarding a request, the proxy checks if image rectification is enabled.
2. It iterates over all messages in the request body, scanning the `content` array of each.
3. For each block where `type` equals `"image"`, it:
   - Counts the removed base64 data characters
   - Checks the next block for an image cache reference (`[Image: source: /path/to/file]`)
   - Replaces the image block with a text block: `"[Image: please use '{skill_name}' skill to read and analyze this image. Image path: /path/to/file.png]"`
4. The modified request is then forwarded to the upstream provider.

If no `type: "image"` blocks are found, the request passes through unchanged.

## Integration with image-analysis Skill

The rectifier is designed to work with the [Image Analysis Skill](./4.7-image-analysis-skill.md). When enabled together:

1. The rectifier strips image data and inserts a prompt telling the model to call `image-analysis`.
2. The model receives the text prompt and executes the `image-analysis` CLI command.
3. The skill processes the actual image file (or clipboard data) using pre-configured vision AI providers.
4. The model receives the vision analysis result and continues its response.

## Notes

- Image rectification only affects requests forwarded through the proxy. Direct API calls are not affected.
- The feature is disabled by default to avoid interfering with multimodal-capable providers.
- Base64 image data can be several megabytes; removing it significantly reduces request body size.
