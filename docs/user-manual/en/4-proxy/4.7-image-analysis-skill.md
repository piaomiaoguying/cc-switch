# 4.7 Image Analysis Skill

## Overview

The Image Analysis Skill (`image-analysis`) is a standalone Python CLI tool for image vision recognition. It supports 30+ AI providers and is compatible with any AI CLI tool that can execute shell commands.

## Installation & Configuration

The skill is located at `skills/image-analysis/` in the project root.

```bash
# Copy the configuration template
cp skills/image-analysis/scripts/config.example.json skills/image-analysis/scripts/config.json

# Edit config.json and add your API keys
```

The `config.json` file has the following structure:

```json
{
  "default_model": "zhipu",
  "providers": {
    "provider-name": {
      "api_key": "your-api-key",
      "model": "model-id",
      "base_url": "https://api.example.com/v1/chat/completions"
    }
  }
}
```

The `config.json` file is git-ignored and will not be committed.

## Usage

```bash
# Command path
python skills/image-analysis/scripts/vision.py analyze [options]
```

### Options

| Option | Description |
|--------|-------------|
| `--image PATH/URL` | Image file path or URL. Can be specified multiple times for comparison. If omitted, reads from macOS clipboard. |
| `--prompt TEXT` | The analysis prompt (required) |
| `--model NAME` | Use a specific provider by name |
| `--fallback` | Try all providers in order, switching on failure. First success is returned. |
| `--thinking` | Enable extended thinking mode (if provider supports it) |
| `--json` | Output result in JSON format with provider name |
| `--show-usage` | Display token usage (prompt/completion/total) |
| `--config PATH` | Path to custom config.json |

### Examples

```bash
# Analyze a local image with fallback
python skills/image-analysis/scripts/vision.py analyze \
  --image screenshot.png \
  --prompt "What error message is shown in this screenshot?" \
  --fallback

# Analyze a network image
python skills/image-analysis/scripts/vision.py analyze \
  --image https://example.com/photo.jpg \
  --prompt "Describe this photo" \
  --model zhipu

# Compare two images
python skills/image-analysis/scripts/vision.py analyze \
  --image before.png \
  --image after.png \
  --prompt "What changed between these two screenshots?" \
  --fallback --json

# Clipboard image with thinking mode
python skills/image-analysis/scripts/vision.py analyze \
  --prompt "Read all text from this image" \
  --thinking --fallback --show-usage
```

## Supported Image Formats

- **Local files:** jpg, jpeg, png, gif, webp, bmp (auto-detected by extension)
- **Network URLs:** Any publicly accessible image URL
- **Clipboard (macOS):** Reads PNG, TIFF, GIF from system clipboard via AppleScript

## Integration with Image Rectifier

The [Image Rectifier](./4.6-image-rectifier.md) and this skill work together:

1. A non-vision model receives a request containing image blocks
2. The rectifier strips the images, inserting: `"[Image: please use 'image-analysis' skill to read and analyze this image. Image path: /tmp/photo.png]"`
3. The model reads this hint and generates a shell command calling `vision.py analyze`
4. The image-analysis skill executes, calling the configured vision AI provider
5. The vision result is returned to the model, which continues processing

This chain allows non-vision models to effectively handle image-based queries without modifying the upstream provider.

## Notes

- The `--fallback` flag is recommended for reliability: if one provider is down, the next one is tried automatically.
- The clipboard feature only works on macOS and requires no image argument.
- The `config.json` contains API keys and is git-ignored. Never commit it.
- All providers use OpenAI-compatible `/v1/chat/completions` endpoints.
